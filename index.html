Why another Torah app?

Most Torah software is either a massive digital shelf, or a basic PDF viewer. TaryagNotes is a mitzvah-native workbench designed for structured limmud: mareh mekomos, focused chazarah, and clean navigation across a finite backbone.

613 mitzvos over a deliberate halachic backbone

Each mitzvah is wired into a carefully chosen backbone: Chumash, Shas, Rambam, Tur, and the 31 Rishonim cited by the Beis Yosef. Not “a folder of files” — a mapped, normalized, predictable corpus.

  • Mitzvah-centric index spanning core sources
  • Axis filters baked into the mitzvah table
  • Designed for psak-oriented research and review

A reader that feels like a shtender, not a PDF viewer

Dark theme, crisp typography, generous rhythm, and a tool-like UI. Sources, mitzvah details, and notes live together — no drifting into random docs.

  • Adjustable font size / line height
  • Mitzvah-linked notes, bookmarks, history
  • One consistent workspace for “mitzvos + seforim”

Daily cycles + journal + chazarah on the same map

Daily cycles (Daf Yomi, Rambam Yomi, Chumash, etc.) connect to the same mitzvah spine. Learning sessions have a place to land — and a path to review.

  • Calendar-aware daily view
  • Session timer + simple streaks
  • Internal flashcards driven by the mitzvah table

How does it work inside?

The corpus is curated and mapped to mitzvos, indexed with SQLite FTS, and surfaced through an Avalonia desktop UI. Only your personal data is writable; the corpus stays read-only and consistent.

Curated, mitzvah-mapped corpus

Not a folder of files — normalized texts mapped to a structured mitzvah table and metadata.

SQLite FTS as a halachic index

Millisecond queries + predictable filters (who/where/when/shiur/trigger/violation) for precision work.

Avalonia desktop workspace

Workbench UI (grid, reader, calendar, notes, status) with private per-profile data storage.
