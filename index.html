How does it work inside?

Under the hood, a curated halachic corpus is mapped onto the 613 mitzvos, indexed with SQLite FTS, and surfaced through an Avalonia desktop UI. Only your data ‚Äì notes, journal, flashcards, preferences ‚Äì is writable. The corpus remains read-only, consistent, and the same on every machine.

Curated, mitzvah-mapped corpus

Static, read-only seforim and mitzvah data ship with the app: Chumash, Shas, Rambam, Tur, and the 31 core Rishonim used by the Beis Yosef. The texts are not ‚Äúa big folder of files‚Äù ‚Äì they are pre-normalized and mapped to a structured mitzvah table and metadata so queries stay sharp and predictable.

SQLite FTS as a halachic index

A SQLite Full-Text Search (FTS) index powers millisecond-level queries across dapim, prakim, and simanim in the curated corpus. Combined with mitzvah IDs and axis filters (time, place, who, shiur, trigger, violation), it behaves less like a general search engine and more like a precision instrument for mareh mekomos.

Avalonia desktop workspace

Avalonia provides a modern, cross-platform shell: mitzvah grid, sefer reader, global toolbar, daily-cycle calendar, notes pane, and status bar. User data ‚Äì notes, journal, SRS progress and preferences ‚Äì lives in a separate SQLite database per profile, so multiple learners can share the same corpus while keeping their learning entirely private.
