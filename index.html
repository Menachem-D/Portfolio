How does it work inside?

Under the hood, a curated halachic corpus is mapped onto the 613 mitzvos, indexed with SQLite FTS, and surfaced through an Avalonia desktop UI. Only your own data â€“ notes, journal, flashcards, preferences â€“ are writable; the corpus itself is read-only.

Curated halachic corpus

Static, read-only seforim and mitzvah data are bundled with the app: Chumash, Shas, Rambam, Tur, and the 31 core Rishonim used by the Beis Yosef. The texts are organized around a structured mitzvah table and metadata, not exposed for editing â€“ only your notes and user database are writable.

SQLite FTS indices

A SQLite Full-Text Search (FTS) index provides millisecond-level queries across dapim, prakim, and simanim in the curated corpus. Combined with mitzvah IDs and axis filters (time, place, gender, shiur, trigger), this turns the app into a precise reference instrument rather than a noisy general search engine.

Avalonia desktop workspace

Avalonia delivers a modern, cross-platform UI: mitzvah grid, sefer reader, global toolbar, daily-cycle calendar, notes pane, and status bar. User data â€“ notes, journal, SRS progress and preferences â€“ lives in a separate SQLite database per user, so multiple profiles can share the same app and corpus while keeping their learning private.
